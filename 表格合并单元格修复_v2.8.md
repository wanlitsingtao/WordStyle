# 表格合并单元格修复说明 - v2.8

## 🎯 问题描述

**当前问题**：转换功能无法正确复制表格中的合并单元格。

**表现**：
- 横向合并的单元格（gridSpan）变成多个独立单元格
- 纵向合并的单元格（vMerge）变成多个独立单元格
- 表格结构被破坏，内容错位

## ❌ 原因分析

### 原有实现的问题

```python
# 旧代码
rows = len(source_table.rows)
cols = len(source_table.columns)
new_table = target_doc.add_table(rows=rows, cols=cols)  # ← 问题所在
```

**问题分析**：
1. `target_doc.add_table(rows, cols)` 创建的是一个**规则表格**
2. 不会保留源表格的合并单元格信息（gridSpan, vMerge）
3. 所有单元格都是独立的，合并关系丢失

### Word 表格合并单元格的 XML 表示

**横向合并（gridSpan）**：
```xml
<w:tc>
  <w:tcPr>
    <w:gridSpan w:val="3"/>  <!-- 横跨3列 -->
  </w:tcPr>
  <w:p>...</w:p>
</w:tc>
<!-- 后面缺少2个 <w:tc>，因为被合并了 -->
```

**纵向合并（vMerge）**：
```xml
<!-- 第一行：起始单元格 -->
<w:tc>
  <w:tcPr>
    <w:vMerge w:val="restart"/>  <!-- 纵向合并开始 -->
  </w:tcPr>
  <w:p>内容</w:p>
</w:tc>

<!-- 第二行：继续单元格 -->
<w:tc>
  <w:tcPr>
    <w:vMerge w:val="continue"/>  <!-- 纵向合并继续 -->
  </w:tcPr>
  <!-- 没有内容，因为与上一行合并 -->
</w:tc>
```

## ✅ 解决方案

### 新策略：深拷贝 XML 元素

不再使用 `add_table()` 创建新表格，而是**直接深拷贝源表格的 XML 元素**，这样可以保留所有属性。

### 核心改进

#### 1. 深拷贝表格 XML

```python
from copy import deepcopy

# 深拷贝表格的 XML 元素，保留所有属性（包括合并单元格）
new_table_elem = deepcopy(source_table._element)

# 将表格添加到目标文档
target_doc._element.body.append(new_table_elem)

# 获取新创建的表格对象
new_table = target_doc.tables[-1]
```

**优势**：
- ✅ 保留 gridSpan（横向合并）
- ✅ 保留 vMerge（纵向合并）
- ✅ 保留所有单元格属性
- ✅ 保留表格结构完整性

#### 2. 智能处理合并单元格

```python
for i, row in enumerate(source_table.rows):
    for j, cell in enumerate(row.cells):
        # 检查单元格属性
        tc = cell._element
        tcPr = tc.find(qn('w:tcPr'))
        
        if tcPr is not None:
            # 检查是否是纵向合并的继续单元格
            vMerge = tcPr.find(qn('w:vMerge'))
            if vMerge is not None and vMerge.get(qn('w:val')) == 'continue':
                continue  # 跳过，这个单元格已经被合并了
            
            # 检查横向合并
            gridSpan = tcPr.find(qn('w:gridSpan'))
            if gridSpan is not None:
                # 这是一个横向合并的起始单元格，保留它
                pass
        
        # 获取对应的目标单元格
        try:
            new_cell = new_table.cell(i, j)
        except IndexError:
            # 如果索引超出范围，说明是合并单元格，跳过
            continue
        
        # 清空并复制内容
        new_cell._element.clear_content()
        # ... 复制段落内容 ...
```

**关键点**：
- ✅ 检测并跳过 `vMerge='continue'` 的单元格
- ✅ 保留 `gridSpan` 的起始单元格
- ✅ 使用 try-except 处理索引越界（合并单元格导致）

#### 3. 添加警告回调

```python
if objects or shapes:
    # 复制特殊对象
    for obj in objects + shapes:
        new_obj = deepcopy(obj)
        new_para._element.append(new_obj)
    
    # 输出警告

        try:
            warning_msg = f"表格 {table_idx} 单元格 [{i},{j}] 包含 OLE/VML 对象"

            pass
```

## ⚠️ 注意事项

### 1. 深拷贝的限制

**优点**：
- ✅ 完整保留表格结构
- ✅ 保留所有单元格属性
- ✅ 保留合并关系

**潜在问题**：
- ⚠️ 可能复制不需要的样式 ID
- ⚠️ 需要后续设置正确的表格样式

**解决**：
```python
# 深拷贝后，重新设置样式
try:
    new_table.style = table_style
except:
    pass
```

### 2. 单元格索引问题

**问题**：
- 合并单元格会导致 `cell(i, j)` 索引不准确
- 例如：横向合并3列，实际只有1个单元格，但索引可能是 (0, 0), (0, 1), (0, 2)

**解决**：
```python
try:
    new_cell = new_table.cell(i, j)
except IndexError:
    # 如果索引超出范围，说明是合并单元格，跳过
    continue
```

### 3. vMerge 的处理

**Word 的 vMerge 机制**：
- `vMerge='restart'`：纵向合并的起始单元格（有内容）
- `vMerge='continue'`：纵向合并的继续单元格（无内容，应跳过）

**处理逻辑**：
```python
vMerge = tcPr.find(qn('w:vMerge'))
if vMerge is not None and vMerge.get(qn('w:val')) == 'continue':
    continue  # 跳过继续单元格
```

### 4. 图片和特殊对象

**当前支持**：
- ✅ 普通图片（a:blip）
- ✅ OLE 对象（w:object）
- ✅ VML 形状（v:shape）

**处理方式**：
- 图片：提取并重新插入
- OLE/VML：深拷贝 XML 元素
