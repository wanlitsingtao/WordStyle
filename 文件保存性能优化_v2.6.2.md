# 文件保存性能优化说明 - v2.6.2

## 🚀 优化目标

解决"再次转换同一个源文件导致效率低，用户等待时间长"的问题。

## ❌ 优化前的问题

### 场景描述

用户操作流程：
1. 第一次转换 `document.docx` → 生成 `document_converted.docx` ✅
2. 用 Word 打开 `document_converted.docx` 查看结果
3. **不关闭 Word**，再次转换同一个源文件
4. 程序尝试保存 `document_converted.docx` → 失败（文件被占用）
5. 程序等待 0.5 秒后重试...
6. 再次失败，再等待 0.5 秒...
7. 最多重试 10 次，每次等待 0.5 秒
8. **总等待时间：最多 5 秒！** 😫

### 问题根源

```python
# 旧逻辑：先尝试原文件名，失败后才使用备用文件名
for attempt in range(10):
    try:
        doc.save(output_file)  # ← 尝试保存，但文件被占用
        return success
    except PermissionError:
        time.sleep(0.5)  # ← 白白等待 0.5 秒
        # 生成备用文件名，下次循环才使用
```

**缺陷**：
- ❌ 明知文件被占用，还要先尝试保存
- ❌ 每次失败都要等待 0.5 秒
- ❌ 最多重试 10 次，浪费大量时间
- ❌ 用户体验差，感觉程序"卡住"了

## ✅ 优化后的方案

### 新策略：预先检测，直接使用备用文件名

```python
# 新逻辑：先检查文件是否被占用，如果是则直接使用备用文件名
if os.path.exists(output_file):
    try:
        with open(output_file, 'r+b') as f:
            pass  # 能打开，说明没被占用
    except (PermissionError, IOError):
        # 文件被占用，直接使用备用文件名
        backup_file = f"{base}_{HHMM}{ext}"
        doc.save(backup_file)  # ← 直接保存，无需等待
        return success
```

### 优势对比

| 项目 | 优化前 | 优化后 |
|------|--------|--------|
| 检测时机 | 保存失败后 | 保存之前 |
| 等待时间 | 最多 5 秒 | 0 秒 |
| 重试次数 | 最多 10 次 | 0 次 |
| 响应速度 | 慢 | 快 ⚡ |
| 用户体验 | 差（感觉卡顿） | 好（立即响应） |

## 🔧 技术实现

### 核心改进

#### 1. 预先检测文件占用状态

```python
# 首先检查文件是否存在且被占用
if os.path.exists(output_file):
    try:
        # 尝试以独占模式打开文件，检查是否被占用
        with open(output_file, 'r+b') as f:
            pass  # 如果能打开，说明没被占用
    except (PermissionError, IOError):
        # 文件被占用，直接使用备用文件名
        base, ext = os.path.splitext(output_file)
        time_suffix = datetime.now().strftime("_%H%M")
        backup_file = f"{base}{time_suffix}{ext}"
        print(f"  检测到文件被占用，直接使用备用文件名: {backup_file}")
        
        # 尝试保存备用文件名
        try:
            doc.save(backup_file)
            return True, backup_file, f"原文件被占用，文档已保存到: {backup_file}"
        except Exception as e:
            return False, output_file, f"保存备用文件失败: {e}"
```

**关键点**：
- ✅ 在保存之前就检测文件状态
- ✅ 如果被占用，立即使用备用文件名
- ✅ 无需等待，无需重试
- ✅ 节省最多 5 秒的等待时间

#### 2. 保留原有的重试机制作为后备

```python
# 文件未被占用或不存在，直接保存
current_file = output_file
for attempt in range(max_retries):
    try:
        doc.save(current_file)
        return True, current_file, "文档已保存"
    except (PermissionError, OSError, IOError) as e:
        # 文件在保存过程中被占用（罕见情况）
        # 生成新的文件名（使用更精确的时间戳）
        time_suffix = datetime.now().strftime("_%H%M%S")
        current_file = f"{base}{time_suffix}{ext}"
        time.sleep(0.3)  # 减少等待时间到 0.3 秒
```

**后备机制**：
- ✅ 处理"保存过程中文件被占用"的罕见情况
- ✅ 使用更精确的时间戳 `_HHMMSS` 避免冲突
- ✅ 减少等待时间从 0.5 秒到 0.3 秒

### 完整流程

```
开始保存
  ↓
文件是否存在？
  ├─ 否 → 直接保存原文件名 ✅
  └─ 是 ↓
      能否打开文件？
        ├─ 能 → 文件未被占用，直接保存原文件名 ✅
        └─ 不能 ↓
            文件被占用！
              ↓
            生成备用文件名（_HHMM）
              ↓
            直接保存备用文件名 ✅
            （无需等待，无需重试）
```

## ⚠️ 注意事项

### 1. 文件占用检测的局限性

**检测方法**：
```python
with open(output_file, 'r+b') as f:
    pass
```

**优点**：
- ✅ 快速（毫秒级）
- ✅ 简单可靠
- ✅ Windows/Linux 通用

**局限**：
- ⚠️ 只能检测当前进程的占用
- ⚠️ 某些特殊情况下可能误判

**应对**：
- 保留原有的重试机制作为后备
- 如果检测失败，仍然会尝试保存并重试

### 2. 时间戳格式

**正常情况**：`_HHMM`（时分）
```
document_converted_1430.docx
```

**重试情况**：`_HHMMSS`（时分秒）
```
document_converted_143025.docx
```

**原因**：
- 正常情况使用简洁格式
- 重试时使用精确格式避免冲突

### 3. 文件管理

**建议**：
- 定期清理带时间戳的备份文件
- 保留最新的转换结果
- 可以使用搜索功能：`*_????_.docx`
