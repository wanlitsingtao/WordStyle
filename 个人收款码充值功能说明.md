## 💡 核心优势

### vs 传统商户号方案

| 特性 | 个人收款码 | 微信商户号 |
|------|-----------|-----------|
| **申请难度** | ✅ 零门槛 | ❌ 需要营业执照 |
| **成本** | ✅ 免费 | ❌ 手续费0.6% |
| **到账速度** | ✅ 即时到账 | ⚠️ T+1结算 |
| **开发复杂度** | ✅ 简单 | ❌ 复杂（需对接SDK） |
| **自动验证** | ⚠️ 半自动 | ✅ 全自动回调 |
| **适用场景** | ✅ 小规模/内部工具 | ✅ 商业化产品 |

### 为什么选择这个方案？

1. **快速上线** - 无需等待商户号审核（通常3-7天）
2. **零成本** - 没有手续费，适合初期验证
3. **简单可靠** - 代码量少，维护成本低
4. **灵活可控** - 可随时调整充值策略

## ⚠️ 注意事项

### 信任机制说明

由于个人收款码无法自动回调，我们采用**基于信任的半自动化方案**：

```
用户扫码支付 → 您收到微信通知 → 用户点击"我已支付" → 系统自动充值
```

**潜在风险：**
- 用户可能点击"我已支付"但未实际支付
- 小额充值影响不大，可作为营销成本
- 发现异常用户可手动清零余额

**建议措施：**
1. ✅ 在用户协议中说明诚信原则
2. ✅ 设置单次充值上限（如¥100）
3. ✅ 定期核对微信收款记录和系统充值记录
4. ✅ 对可疑用户进行人工审核

## 🆘 常见问题

### Q1: 用户上传了错误的收款码怎么办？
**A:** 删除 `personal_qr_code.png` 文件，重新上传正确的即可。或者在网页中重新上传覆盖。

### Q2: 用户点击"我已支付"但实际没支付怎么办？
**A:** 
- 小额充值影响不大，可作为营销成本
- 发现异常用户可手动编辑 `user_data.json` 清零余额
- 建议在用户协议中说明诚信原则

### Q3: 能否实现真正的自动验证？
**A:** 
- 个人收款码无法实现自动回调
- 如需自动验证，需申请微信支付商户号（需要营业执照）
- 或使用第三方聚合支付平台（如收钱吧、Ping++）

### Q4: 收款码会过期吗？
**A:** 
- 微信个人收款码长期有效
- 建议每半年更新一次以确保安全

### Q5: 支持支付宝收款码吗？
**A:** 
- 当前仅支持微信
- 如需支持支付宝，可添加第二个收款码文件
- 让用户通过单选框选择支付方式

### Q6: 如何防止用户重复点击"我已支付"？
**A:** 
- 当前设计：每次生成新订单，订单号唯一
- 后端会检查订单状态，防止重复充值
- 可以添加订单有效期（如15分钟）
