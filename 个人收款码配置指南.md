# 📱 个人微信收款码配置指南

## ✅ 功能说明

现在系统支持使用您的**个人微信收款码**进行充值！用户扫码支付后，点击"我已支付"按钮即可自动充值。

---

## 🔧 配置步骤（3步完成）

### 步骤1：获取您的微信收款码

1. **打开微信** → 点击右上角"+"
2. 选择 **"收付款"**
3. 选择 **"二维码收款"**
4. 点击 **"保存收款码"**
5. 将图片保存到电脑

![微信收款码获取步骤](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/7078644961/p852435.png)

---

### 步骤2：重命名并放置文件

1. 将收款码图片重命名为：**`personal_qr_code.png`**
2. 放到项目根目录：**`E:\LingMa\WordStyle\personal_qr_code.png`**

**或者支持以下格式：**
- `personal_qr_code.png`（推荐）
- `personal_qr_code.jpg`
- `personal_qr_code.jpeg`

---

### 步骤3：重启应用

```bash
# 重启Streamlit应用
cd E:\LingMa\WordStyle
.venv\Scripts\python.exe -m streamlit run app.py --server.port=8501
```

**或者双击运行：**
```
启动转换工具.bat
```

## ⚠️ 重要说明

### 优点
✅ **简单快捷** - 无需申请商户号  
✅ **零成本** - 个人收款码免费使用  
✅ **即时到账** - 款项直接进入您的微信  
✅ **自动化** - 用户自助充值，无需人工干预  

### 注意事项
⚠️ **信任机制** - 基于用户诚实点击"我已支付"  
⚠️ **无自动验证** - 无法自动检测是否真的支付  
⚠️ **建议监控** - 定期检查微信收款记录  

### 防作弊建议

#### 方案1：定期核对（推荐）
```
每周查看微信收款记录，对比系统充值记录
发现异常可手动调整用户余额
```

#### 方案2：添加备注验证
```
要求用户在支付时添加备注（订单号）
您可以选择性验证大额充值
```

#### 方案3：设置充值上限
```
单次最高充值 ¥100
每日最多充值 3 次
防止恶意刷充值
```

## 🛠️ 高级配置（可选）

### 修改收款码路径

编辑 `app.py` 第511行：
```python
qr_code_path = "personal_qr_code.png"  # 修改为您的路径
```

### 添加充值限制

在 `app.py` 中添加检查逻辑：
```python
# 检查今日充值次数
today_recharges = get_today_recharge_count(st.session_state.user_id)
if today_recharges >= 3:
    st.error("❌ 今日充值次数已达上限")
    return
```

### 添加管理员审核

对于大额充值（>¥50），可以要求管理员审核：
```python
if amount > 50:
    st.warning("⚠️ 大额充值需要管理员审核")
    st.info("请联系管理员确认后手动充值")
else:
    # 正常充值流程
    ...
```

## 🆘 常见问题

### Q1: 用户上传了错误的收款码怎么办？
**A:** 删除 `personal_qr_code.png` 文件，重新上传正确的即可。

### Q2: 用户点击"我已支付"但实际没支付怎么办？
**A:** 
- 小额充值影响不大，可作为营销成本
- 发现异常用户可手动清零余额
- 建议在用户协议中说明诚信原则

### Q3: 能否实现真正的自动验证？
**A:** 
- 个人收款码无法实现自动回调
- 如需自动验证，需申请微信支付商户号
- 或使用第三方聚合支付平台（如收钱吧）

### Q4: 收款码会过期吗？
**A:** 
- 微信个人收款码长期有效
- 建议每半年更新一次以确保安全

### Q5: 支持支付宝收款码吗？
**A:** 
- 当前仅支持微信
- 如需支持支付宝，可添加第二个收款码文件
- 让用户选择支付方式
