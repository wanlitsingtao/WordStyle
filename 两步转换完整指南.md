# 两步转换完整指南 - Visio 图 + 所有功能

## 🎯 问题说明

您发现使用 Word COM 转换后：
- ✅ Visio 图完整保留
- ❌ 样式没有应用
- ❌ 祈使语气没有转换
- ❌ 应答句没有插入

**原因**：COM 转换只是"另存为"，不做其他处理。

## 💡 解决方案

### 方案 A：使用自动化脚本（推荐）

#### 一键完成所有转换

```bash
python two_step_conversion.py
```

这个脚本会自动：
1. **第一步**：COM 转换保留 Visio 图
2. **第二步**：普通转换应用样式、祈使语气、应答句等
3. **清理**：删除临时文件

#### 使用方法

```bash
# 运行脚本
python two_step_conversion.py

# 按提示选择是否进行祈使语气转换
是否进行祈使语气转换？(y/n，默认y): y

# 等待完成...
```

#### 输出文件

- `8js3_temp_com.docx` - 中间文件（有 Visio 图，会被删除）
- `8js3_final.docx` - 最终文件（有所有功能，Visio 图为占位文本）

### 方案 B：GUI 手动两步操作

#### 第一步：COM 转换

1. 启动 GUI
2. 选择源文件 `8js3.docx`
3. 勾选 **"使用 Word COM 转换"**
4. 点击"开始转换"
5. 生成 `8js3_converted.docx`（有 Visio 图）

#### 第二步：普通转换

1. 清除文件列表或重启 GUI
2. 选择源文件为 `8js3_converted.docx`
3. **取消勾选** "使用 Word COM 转换"
4. 勾选需要的功能（祈使语气、应答句等）
5. 点击"开始转换"
6. 生成 `8js3_converted_converted.docx`

#### 第三步：处理 Visio 图

最终文件中 Visio 图位置会显示占位文本，需要：
1. 打开 `8js3_converted.docx`（有 Visio 图）
2. 打开 `8js3_converted_converted.docx`（有占位文本）
3. 手动复制粘贴 Visio 图

## 📊 两种方案对比

| 特性 | 自动化脚本 | GUI 手动操作 |
|------|-----------|-------------|
| 操作步骤 | 1 步 | 3 步 |
| 自动化程度 | 高 | 低 |
| 需要手动复制 Visio | ❌ 不需要* | ✅ 需要 |
| 灵活性 | 中等 | 高 |
| 适合场景 | 批量处理 | 精细调整 |

\* 脚本会保留 Visio 图在最终文件中，但 python-docx 会将其替换为占位文本。如果需要 Visio 图，仍需从中间文件复制。

## 🔧 自定义脚本参数

编辑 `two_step_conversion.py` 底部：

```python
if __name__ == "__main__":
    # 修改这些参数
    source_file = "e:\\LingMa\\your_source.docx"
    template_file = "e:\\LingMa\\your_template.docx"
    output_file = "e:\\LingMa\\your_output.docx"
```

或者在代码中修改默认值：

```python
success = two_step_conversion(
    source_file=source_file,
    template_file=template_file,
    output_file=output_file,
    do_mood=True,  # 是否祈使语气转换

    list_bullet="● "

```

## ⚠️ 重要说明

### 关于 Visio 图的最终状态

无论使用哪种方案，**最终文件中 Visio 图的处理方式**：

1. **中间文件** (`*_temp_com.docx` 或 `*_converted.docx`)
   - ✅ 包含完整的 Visio 图
   - ✅ 可以正常编辑
   - ⚠️ 样式可能未应用

2. **最终文件** (`*_final.docx` 或 `*_converted_converted.docx`)
   - ✅ 样式已应用
   - ✅ 祈使语气已转换
   - ✅ 应答句已插入
   - ⚠️ Visio 图位置显示占位文本

### 为什么最终文件没有 Visio 图？

因为第二步使用的是 `doc_converter.py`（python-docx），它会：
- 检测到 OLE 对象
- 跳过复制（避免文档损坏）
- 插入占位文本 `[此处有 Visio 图或 OLE 对象，请手动复制]`

这是**目前的技术限制**。

### 如何获得完美的最终文件？

**唯一方法**：手动复制 Visio 图

1. 从中间文件复制 Visio 图
2. 粘贴到最终文件的对应位置
3. 删除占位文本

对于 69 个 Visio 图，预计需要 30-60 分钟。
