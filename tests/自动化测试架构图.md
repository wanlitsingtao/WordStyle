## 🔄 测试执行流程

```
启动测试
   │
   ├─→ pytest 发现测试文件
   │      │
   │      ├─→ conftest.py 加载配置
   │      │      └─→ 初始化 fixtures
   │      │
   │      └─→ 收集所有 test_*.py
   │
   ├─→ 执行测试
   │      │
   │      ├─→ API 测试
   │      │      ├─→ requests 发送HTTP请求
   │      │      ├─→ 验证响应状态码
   │      │      └─→ 验证响应数据
   │      │
   │      ├─→ UI 测试
   │      │      ├─→ Playwright 启动浏览器
   │      │      ├─→ 模拟用户操作
   │      │      └─→ 验证页面状态
   │      │
   │      └─→ 性能测试
   │             ├─→ 并发请求
   │             ├─→ 计时统计
   │             └─→ 性能指标计算
   │
   └─→ 生成报告
          ├─→ 控制台输出
          ├─→ HTML报告 (可选)
          └─→ JUnit XML (可选)
```

## 🎯 关键设计原则

### 1. 测试隔离
```python
# 每个测试独立,不依赖其他测试
@pytest.fixture
def test_user():
    user_id = f"test_{int(time.time())}"
    # 创建临时用户
    yield user_id
    # 清理(可选)
```

### 2. 标记分类
```python
@pytest.mark.slow  # 耗时测试
def test_full_workflow():
    ...

# 运行快速测试
pytest tests/ -m "not slow"
```

### 3. 参数化测试
```python
@pytest.mark.parametrize("users", [5, 10, 20, 50])
def test_concurrent(users):
    ...
```

### 4. 重试机制
```python
@pytest.mark.flaky(reruns=3)
def test_unstable_feature():
    ...
```

## 📈 扩展性设计

```
当前架构支持轻松扩展:

1. 添加新API测试
   └─→ 在 test_api.py 中添加新类

2. 添加新UI测试
   └─→ 在 test_ui.py 中添加新方法

3. 添加新性能场景
   └─→ 在 test_performance.py 中添加

4. 集成CI/CD
   └─→ 使用 pytest --junitxml

5. 自定义报告
   └─→ 使用 pytest-html 插件
```
