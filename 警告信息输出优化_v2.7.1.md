# 警告信息输出优化说明 - v2.7.1

## 🎯 问题描述

用户报告以下重要警告信息没有输出到 GUI 日志中：

```
[WARNING] 段落 168 包含 OLE/VML 对象，暂时跳过以避免文档损坏
  - OLE 对象数: 0
  - VML 形状数: 1
检测到文件被占用，直接使用备用文件名: E:/LingMa/mj_converted_2233.docx
```

这些消息只输出到控制台（print），GUI 用户看不到。

## ❌ 原因分析

### 问题根源

代码中使用 `print()` 输出警告信息：

```python
# doc_converter.py 第 482-484 行
print(f"[WARNING] 段落 {para_idx} 包含 OLE/VML 对象，暂时跳过以避免文档损坏")
print(f"  - OLE 对象数: {len(has_ole_objects)}")
print(f"  - VML 形状数: {len(has_vml_shapes)}")

# doc_converter.py 第 1219 行
print(f"  检测到文件被占用，直接使用备用文件名: {backup_file}")
```

**问题**：
- `print()` 输出到标准输出（控制台）
- GUI 程序运行时没有控制台窗口
- 用户看不到这些重要的警告信息

## ✅ 解决方案

### 方案：添加 warning_callback 回调机制

通过回调函数将警告信息传递到 GUI 日志中显示。

### 实现步骤

#### 1. 修改方法签名，添加 warning_callback 参数

**修改的方法**：
- `copy_paragraph_with_images()` - 接收 warning_callback
- `copy_element_with_objects()` - 接收并传递 warning_callback
- `convert_styles()` - 接收并传递 warning_callback
- `full_convert()` - 接收并传递 warning_callback

#### 2. 修改警告输出逻辑

**修改前**：
```python
if has_ole_objects or has_vml_shapes:
    print(f"[WARNING] 段落 {para_idx} 包含 OLE/VML 对象...")
    print(f"  - OLE 对象数: {len(has_ole_objects)}")
    print(f"  - VML 形状数: {len(has_vml_shapes)}")
```

**修改后**：
```python
if has_ole_objects or has_vml_shapes:
    warning_msg = f"[WARNING] 段落 {para_idx} 包含 OLE/VML 对象...\n  - OLE 对象数: ...\n  - VML 形状数: ..."
    print(warning_msg)  # 仍然输出到控制台（调试用）
    
    # 如果有回调函数，也通过回调输出
    if warning_callback:
        try:
            warning_callback(warning_msg)
        except:
            pass
```

#### 3. GUI 中定义 warning_callback

**单文件模式**：
```python
def warning_callback(message):
    def update_ui():
        self.log(f"  ⚠ {message}")
    self.root.after(0, update_ui)
```

**多文件模式**：
```python
def warning_callback(message):
    def update_ui():
        self.log(f"    ⚠ {message}")
    self.root.after(0, update_ui)
```

#### 4. 传递 warning_callback 到 full_convert

```python
success, actual_file, msg = self.converter.full_convert(
    source_file=source,
    template_file=self.template_file.get(),
    output_file=output,
    # ... 其他参数 ...
    progress_callback=progress_callback,
    warning_callback=warning_callback  # ← 新增
)
```

## 🔧 技术实现

### 调用链

```
GUI do_conversion()
  ↓ 定义 warning_callback
  
  ↓ 调用 full_convert(warning_callback=...)
  
  ↓ full_convert 调用 convert_styles(warning_callback=...)
  
  ↓ convert_styles 调用 copy_paragraph_with_images(warning_callback=...)
  
  ↓ copy_paragraph_with_images 检测到 OLE/VML
  
  ↓ 调用 warning_callback(message)
  
  ↓ GUI 的 warning_callback 执行
  
  ↓ self.log(f"  ⚠ {message}")
  
  ↓ 用户看到警告信息 ✅
```

## ⚠️ 注意事项

### 1. 保留 print() 输出

**原因**：
- 方便命令行调试
- 保留原始行为
- 开发者可以看到详细信息

**实现**：
```python
print(warning_msg)  # 仍然输出到控制台

if warning_callback:
    warning_callback(warning_msg)  # 同时输出到 GUI
```

### 2. 异常处理

**保护机制**：
```python
if warning_callback:
    try:
        warning_callback(warning_msg)
    except:
        pass  # 如果回调失败，不影响主流程
```

**原因**：
- 避免回调异常导致转换失败
- 保证核心功能不受影响

### 3. 线程安全

**使用 root.after()**：
```python
def warning_callback(message):
    def update_ui():
        self.log(f"  ⚠ {message}")
    self.root.after(0, update_ui)  # ← 确保在主线程执行
```

**原因**：
- 转换在后台线程执行
- GUI 更新必须在主线程
- `after(0)` 确保线程安全

### 4. "文件被占用"警告

**当前状态**：
- `save_with_retry()` 中的 `print()` 仍然存在
- 但没有通过 warning_callback 传递

**原因**：
- `save_with_retry()` 是底层方法，不直接接收 warning_callback
- 需要在更高层包装

**改进建议**：
- 可以在 `insert_response_after_headings()` 中捕获这个消息
- 或者在 `full_convert()` 中包装 save_with_retry 的调用
