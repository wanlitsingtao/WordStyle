# 表格丢失问题修复说明 - v2.8.1

## 🐛 问题描述

**v2.8 版本的严重 Bug**：改进合并单元格复制后，转换后的文档中**表格全部丢失**。

### 用户反馈

> "改进正确复制表格合并单元格后，程序运行结果，转换后的文档中没有表格，表格全部丢失。"

## ❌ 原因分析

### v2.8 的实现方式（有问题）

```python
# 深拷贝整个表格 XML
new_table_elem = deepcopy(source_table._element)

# 直接添加到目标文档
target_doc._element.body.append(new_table_elem)

# 获取新表格
new_table = target_doc.tables[-1]
```

### 问题根源

#### 1. 关系 ID 冲突

深拷贝的表格 XML 中包含：
- 图片的 `r:embed` 引用（如 `rId5`）
- 样式的引用
- 其他资源的引用

这些 rId 在源文档中有效，但在目标文档中：
- 可能不存在对应的资源
- 可能与目标文档的 rId 冲突
- 导致 Word 无法解析表格

#### 2. 段落引用问题

表格内的段落元素可能包含：
- 指向源文档的关系
- 特定的样式 ID
- 编号列表的引用

这些都可能在目标文档中失效。

#### 3. Word 解析失败

当 Word 打开文档时：
1. 发现表格 XML 中有无效的 rId 引用
2. 认为表格结构损坏
3. **直接丢弃整个表格**
4. 用户看到的结果：表格消失

## ✅ 修复方案

### 新策略：分两步处理

**第一步**：创建规则表格并复制合并结构  
**第二步**：填充单元格内容

### 核心改进

#### 1. 使用 add_table() 创建表格

```python
# 创建规则表格（确保表格能被 Word 识别）
rows = len(source_table.rows)
cols = len(source_table.columns)
new_table = target_doc.add_table(rows=rows, cols=cols)
```

**优势**：
- ✅ 表格结构与目标文档兼容
- ✅ 没有无效的关系引用
- ✅ Word 能正常解析

#### 2. 手动复制合并单元格属性

```python
# 遍历所有单元格，复制合并属性
for i in range(rows):
    for j in range(cols):
        source_cell = source_table.cell(i, j)
        new_cell = new_table.cell(i, j)
        
        tc = source_cell._element
        tcPr = tc.find(qn('w:tcPr'))
        
        if tcPr is not None:
            # 复制 gridSpan（横向合并）
            gridSpan = tcPr.find(qn('w:gridSpan'))
            if gridSpan is not None:
                new_tc = new_cell._element
                new_tcPr = new_tc.find(qn('w:tcPr'))
                if new_tcPr is None:
                    new_tcPr = OxmlElement('w:tcPr')
                    new_tc.insert(0, new_tcPr)
                
                new_gridSpan = deepcopy(gridSpan)
                new_tcPr.append(new_gridSpan)
            
            # 复制 vMerge（纵向合并）
            vMerge = tcPr.find(qn('w:vMerge'))
            if vMerge is not None:
                # ... 类似的处理 ...
```

**关键点**：
- ✅ 只复制合并属性（gridSpan, vMerge）
- ✅ 不复制整个单元格 XML
- ✅ 避免关系 ID 冲突

#### 3. 填充单元格内容

```python
# 第二步：复制单元格内容
for i, row in enumerate(source_table.rows):
    for j, cell in enumerate(row.cells):
        # 跳过纵向合并的继续单元格
        tc = cell._element
        tcPr = tc.find(qn('w:tcPr'))
        if tcPr is not None:
            vMerge = tcPr.find(qn('w:vMerge'))
            if vMerge is not None and vMerge.get(qn('w:val')) == 'continue':
                continue
        
        new_cell = new_table.cell(i, j)
        new_cell._element.clear_content()
        
        # 复制段落、文本、图片等
        for para in cell.paragraphs:
            new_para = new_cell.add_paragraph()
            # ... 复制内容 ...
```

## ⚠️ 注意事项

### 1. 为什么不用 deepcopy 整个表格？

**原因**：
- ❌ 会复制无效的关系 ID
- ❌ 会复制源文档的特定引用
- ❌ Word 无法解析，导致表格丢失

**正确做法**：
- ✅ 只复制合并属性（gridSpan, vMerge）
- ✅ 重新创建内容（文本、图片）
- ✅ 确保与目标文档兼容

### 2. 合并单元格的检测

**横向合并（gridSpan）**：
```xml
<w:tcPr>
  <w:gridSpan w:val="3"/>  <!-- 横跨3列 -->
</w:tcPr>
```

**纵向合并（vMerge）**：
```xml
<!-- 起始单元格 -->
<w:tcPr>
  <w:vMerge w:val="restart"/>
</w:tcPr>

<!-- 继续单元格（应跳过） -->
<w:tcPr>
  <w:vMerge w:val="continue"/>
</w:tcPr>
```

### 3. 异常处理

```python
try:
    source_cell = source_table.cell(i, j)
    new_cell = new_table.cell(i, j)
    # ... 处理 ...
except Exception as e:
    # 合并单元格可能导致索引问题
    if warning_callback:
        warning_callback(f"表格 {table_idx} 单元格 [{i},{j}] 处理失败: {e}")
```

### 4. 性能考虑

**两遍遍历**：
- 第一遍：复制合并属性（rows × cols）
- 第二遍：复制内容（实际单元格数）

**影响**：
- 对于大表格，可能需要更多时间
- 但保证了正确性和兼容性
