## 🔧 技术实现

### 1. 配置获取函数

**位置**: [app.py](file://e:\LingMa\WordStyle\app.py#L341-L351)

```python
def get_free_paragraphs_config():
    """从后端获取免费额度配置"""
    try:
        import requests
        response = requests.get(f"{BACKEND_URL}/api/admin/config/free-paragraphs", timeout=2)
        if response.status_code == 200:
            data = response.json()
            return data.get('config_value', 10000)
        return 10000
    except:
        return 10000  # 默认值
```

**功能**：
- 从后端 API 获取免费额度配置
- 如果后端不可用，返回默认值 10,000
- 超时时间设置为 2 秒，避免阻塞页面加载

## ⚠️ 注意事项

### 1. 性能影响

**每次页面加载都会调用 API**：
- 优点：确保配置实时同步
- 缺点：增加一次网络请求

**优化建议**：
- 缓存配置结果（例如缓存 5 分钟）
- 使用 localStorage 存储配置
- 仅在配置变更时重新获取

**示例优化代码**：
```python
import streamlit as st

@st.cache_data(ttl=300)  # 缓存 5 分钟
def get_free_paragraphs_config_cached():
    return get_free_paragraphs_config()

free_quota = get_free_paragraphs_config_cached()
```

---

### 2. 超时处理

**当前超时设置**：2 秒

**可能的问题**：
- 网络慢时可能超时
- 后端响应慢时可能超时

**建议**：
- 根据实际网络情况调整超时时间
- 添加重试机制
- 记录超时日志

---

### 3. 配置一致性

**需要确保**：
- 后端配置的修改立即生效
- 前端获取的是最新配置
- 多个用户看到的是相同的配置

**潜在问题**：
- 后端配置缓存导致延迟
- CDN 缓存导致不同步
- 负载均衡导致配置不一致

**解决方案**：
- 后端配置不使用缓存
- 清除 CDN 缓存
- 使用统一的配置中心

---

### 4. 示例计算的边界情况

**当前示例**：固定减去 3,000

**可能的问题**：
- 如果免费额度 < 3,000，会出现负数
- 例如：额度为 2,000，示例显示"剩余 -1,000 段"

**修复建议**：
```python
example_used = 3000
example_remaining = max(0, int(free_quota) - example_used)

f"例如：今天用了 {example_used:,} 段，剩余 {example_remaining:,} 段"
```
