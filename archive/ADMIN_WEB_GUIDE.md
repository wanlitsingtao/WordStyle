# WordStyle Pro - Web管理后台使用指南

## 📖 概述

`admin_web.py` 是基于 **PostgreSQL数据库** 的完整Web管理后台，替代了旧的基于JSON文件的 `admin_dashboard.py`。

### ✨ 主要特性

- ✅ **实时数据**：直接连接PostgreSQL，显示真实的生产数据
- ✅ **完整功能**：用户管理、任务管理、订单管理、系统配置
- ✅ **数据看板**：可视化统计图表和关键指标
- ✅ **操作便捷**：支持搜索、筛选、分页、批量操作
- ✅ **响应式设计**：适配各种屏幕尺寸

---

## 🚀 快速开始

### 方式1：使用启动脚本（推荐）

双击运行：
```
启动管理后台.bat
```

### 方式2：命令行启动

```bash
# Windows
.venv\Scripts\python.exe -m streamlit run admin_web.py --server.port=8503

# Linux/Mac
source .venv/bin/activate
streamlit run admin_web.py --server.port=8503
```

### 访问地址

打开浏览器访问：**http://localhost:8503**

## 📊 功能模块详解

### 1️⃣ 数据看板

**功能**：展示系统整体运营数据

#### 关键指标卡片

| 指标 | 说明 | 颜色 |
|------|------|------|
| 👥 总用户数 | 注册用户总数 | 紫色渐变 |
| 📝 总转换任务 | 历史转换任务总数 | 粉色渐变 |
| 💰 总收入 | 所有已支付订单金额总和 | 蓝色渐变 |
| ✅ 成功率 | 已完成任务占比 | 绿色渐变 |

#### 任务状态分布

- ⏳ 待处理 (PENDING)
- 🔄 进行中 (PROCESSING)
- ✅ 已完成 (COMPLETED)
- ❌ 失败 (FAILED)

#### 最近活动

三个标签页：
- **最近用户**：显示最新注册的10个用户
- **最近任务**：显示最新的10个转换任务
- **最近订单**：显示最新的10个订单

---

### 2️⃣ 用户管理

**功能**：查看和管理所有用户信息

#### 搜索和筛选

- 🔍 **搜索框**：支持按微信OpenID、昵称、用户名搜索
- 📊 **排序方式**：注册时间、剩余段落、余额
- 📄 **显示数量**：20/50/100条

#### 用户列表字段

| 字段 | 说明 |
|------|------|
| 用户ID | UUID格式的唯一标识 |
| 微信OpenID | 微信用户的唯一标识 |
| 昵称 | 微信昵称或用户名 |
| 剩余段落 | 当前可用的免费段落数 |
| 余额 | 账户余额（元） |
| 状态 | 活跃/禁用 |
| 注册时间 | 用户创建时间 |

#### 用户操作

输入完整的用户ID后，可以执行以下操作：

1. **调整剩余段落**
   - 增加或减少用户的免费段落数
   - 步长：100段
   - 适用场景：补偿用户、赠送活动等

2. **调整余额**
   - 修改用户账户余额
   - 步长：1.0元
   - 适用场景：手动充值、退款等

3. **更新账户状态**
   - 活跃 ↔ 禁用
   - 适用场景：违规用户封禁、账号恢复

**示例操作流程**：
```
1. 在用户列表中找到目标用户
2. 复制该用户的"用户ID"
3. 粘贴到"输入用户ID进行操作"框
4. 修改需要的参数
5. 点击对应的"保存"按钮
6. 系统自动刷新，显示更新后的数据
```

### 3️⃣ 转换任务管理

**功能**：监控和管理所有文档转换任务

#### 筛选条件

- **任务状态**：全部/PENDING/PROCESSING/COMPLETED/FAILED
- **筛选日期**：按创建日期筛选
- **显示数量**：20/50/100条

#### 任务列表字段

| 字段 | 说明 |
|------|------|
| 任务ID | 任务的唯一标识 |
| 用户ID | 发起任务的用户ID（前8位） |
| 文件名 | 上传的文档名称 |
| 段落数 | 文档包含的段落数量 |
| 费用 | 转换消耗的费用（元） |
| 状态 | ⏳待处理/🔄进行中/✅已完成/❌失败 |
| 进度 | 转换进度百分比 |
| 错误信息 | 失败时的错误描述 |
| 创建时间 | 任务创建时间 |
| 完成时间 | 任务完成时间 |

#### 任务操作

输入完整的任务ID后，可以执行：

1. **标记为完成**
   - 将任务状态改为 COMPLETED
   - 进度设置为 100%
   - 记录完成时间
   - 适用场景：任务卡住时手动完成

2. **标记为失败**
   - 将任务状态改为 FAILED
   - 适用场景：任务异常时手动终止

**注意**：
- 这些操作不会退还用户已扣除的段落数
- 如需退还，需要手动在"用户管理"中调整

---

### 4️⃣ 订单管理

**功能**：查看和管理所有充值订单

#### 筛选条件

- **订单状态**：全部/PENDING/PAID/FAILED/REFUNDED
- **筛选日期**：按创建日期筛选
- **显示数量**：20/50/100条

#### 订单列表字段

| 字段 | 说明 |
|------|------|
| 订单号 | 订单的唯一标识 |
| 用户ID | 下单用户ID（前8位） |
| 金额 | 充值金额（元） |
| 段落数 | 购买的段落数量 |
| 套餐 | 套餐名称（如"基础版"） |
| 支付方式 | WECHAT/ALIPAY |
| 状态 | ⏳待支付/✅已支付/❌失败/↩️已退款 |
| 交易号 | 第三方支付平台的交易号 |
| 创建时间 | 订单创建时间 |
| 支付时间 | 实际支付时间 |

#### 订单操作

输入完整的订单号后，可以执行：

1. **标记为已支付**（仅待支付订单）
   - 将订单状态改为 PAID
   - 记录支付时间
   - **自动给用户增加对应段落数**
   - 适用场景：支付回调失败时手动确认

2. **标记为退款**（仅已支付订单）
   - 将订单状态改为 REFUNDED
   - **自动扣除用户对应段落数**
   - 适用场景：用户申请退款

**重要提示**：
- 标记为已支付时，系统会自动增加用户段落数
- 标记为退款时，系统会自动扣除用户段落数
- 请谨慎操作，避免重复增减

### 5️⃣ 系统配置

**功能**：管理系统全局配置项

#### 免费额度配置

**配置项**：`free_paragraphs_on_first_login`

- **说明**：新用户首次微信扫码登录后赠送的免费段落数
- **默认值**：10000段
- **调整步长**：1000段
- **最小值**：0

**操作步骤**：
```
1. 在"首次登录赠送段落数"输入框中输入新数值
2. 点击"💾 保存配置"按钮
3. 系统提示"配置已更新"
4. 新用户注册时将使用新的额度
```

**注意事项**：
- 此配置只影响**新用户首次登录**
- 老用户不受影响
- 不是每天登录都赠送，仅首次

#### 所有配置项列表

显示系统中所有的配置项，包括：
- 配置键（config_key）
- 配置值（config_value）
- 说明（description）
- 更新时间（updated_at）

## 🔧 技术架构

### 数据流

```
用户浏览器
    ↓ HTTP请求
Streamlit服务器 (admin_web.py)
    ↓ SQLAlchemy ORM
PostgreSQL数据库
    ↓ 返回数据
Streamlit渲染界面
    ↓ HTML/CSS/JS
用户浏览器显示
```

### 数据库表结构

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户表 | id, wechat_openid, paragraphs_remaining, balance, is_active |
| conversion_tasks | 转换任务表 | task_id, user_id, filename, paragraphs, status, progress |
| orders | 订单表 | order_no, user_id, amount, paragraphs, status, paid_at |
| system_config | 系统配置表 | config_key, config_value, description |

### 依赖库

- **Streamlit**: Web UI框架
- **SQLAlchemy**: ORM数据库操作
- **Psycopg2**: PostgreSQL驱动

## ⚠️ 注意事项

### 数据安全

1. **不要公开暴露管理后台**
   - 仅在本地或内网访问
   - 如需公网访问，必须添加身份验证

2. **谨慎操作用户数据**
   - 调整余额和段落数前请确认
   - 建议先备份数据库

3. **定期备份数据库**
   ```bash
   # PostgreSQL备份命令
   pg_dump -U postgres wordstyle_db > backup_$(date +%Y%m%d).sql
   ```

### 性能优化

1. **大数据量时的建议**
   - 限制每次显示的记录数（20/50/100）
   - 使用筛选条件缩小范围
   - 避免一次性加载全部数据

2. **查询优化**
   - 已添加索引的字段：user_id, task_id, order_no, created_at
   - 搜索时使用模糊匹配，速度较慢

### 常见问题

**Q1: 看不到任何数据？**
- 检查PostgreSQL是否正常运行
- 检查数据库连接配置（backend/.env）
- 确认数据库中已有数据

**Q2: 修改配置后不生效？**
- 点击"保存配置"按钮
- 刷新页面查看最新数据
- 检查是否有权限写入数据库

**Q3: 操作后数据未更新？**
- 点击浏览器刷新按钮
- 或使用页面的"st.rerun()"自动刷新
- 检查数据库事务是否提交成功
