# 功能增强说明 - v2.5

## 🎉 本次更新内容

### 1. ✅ 应答句插入作为可选项

**新增功能**：
- 添加"插入应答句"复选框
- 未选中时，应答句文本框和样式下拉列表灰化不可操作
- 选中后，控件变为有效，应答句插入功能起效

**界面位置**：
```
┌─────────────────────────────────────┐
│       转换选项                        │
│                                     │
│  ☑ 进行祈使语气转换                  │
│                                     │
│  ☑ 插入应答句  ← 新增复选框          │
│  应答句文本: [_______________]       │
│  应答句样式: [应答句 ▼]              │
└─────────────────────────────────────┘
```

**使用场景**：
- **需要应答句**：勾选复选框，配置文本和样式
- **不需要应答句**：取消勾选，跳过此步骤，加快转换速度

### 2. ✅ 加载文档时灰化按钮

**改进前**：
- 加载文档时按钮仍然可用
- 用户可能在加载过程中点击转换
- 导致错误或混乱

**改进后**：
- 加载开始时自动灰化所有操作按钮
- 加载完成后恢复按钮可用状态
- 防止用户在加载过程中误操作

**灰化的按钮**：
- 开始转换
- 清空日志
- 打开输出文件夹
- 浏览源文件
- 多选源文件
- 浏览模板文件

### 3. ✅ 合并列表编号清理逻辑

**从新脚本合并的功能**：
- 添加 `clean_list_numbering()` 函数
- 专门用于清理列表项开头的数字编号
- 支持格式：`1、`、`1)`、`(1)` 等

**应用位置**：
- 段落中的列表（带自动编号的段落）
- 表格单元格中的列表

**效果对比**：

| 源文本 | 转换前 | 转换后 |
|--------|--------|--------|
| `1、第一项` | ■ 1、第一项 | ● 第一项 |
| `2) 第二项` | ■ 2) 第二项 | ● 第二项 |
| `(3) 第三项` | ■ (3) 第三项 | ● 第三项 |

def toggle_answer_controls(self):
    if self.do_answer_insertion.get():
        self.answer_text_entry.config(state='normal')
        self.answer_style_combo.config(state='readonly')
    else:
        self.answer_text_entry.config(state='disabled')
        self.answer_style_combo.config(state='disabled')

def clean_list_numbering(text):
    """清理开头数字编号：1、 1） (1) 等"""
    pattern = r'^\s*(?:\d+[、\)）]|[（(]\d+[）\)])\s*'
    cleaned = re.sub(pattern, '', text, count=1)
    return cleaned

## 💡 使用建议

### 应答句插入

**何时勾选**：
- ✅ 投标文件需要应答句
- ✅ 文档规范要求统一应答
- ✅ 需要自动化插入固定文本

**何时取消**：
- ❌ 普通文档不需要应答
- ❌ 需要手动编写应答内容
- ❌ 追求更快的转换速度

### 按钮灰化

**正常流程**：
1. 选择文件 → 按钮可用
2. 点击浏览 → 按钮灰化
3. 加载完成 → 按钮恢复
4. 点击转换 → 正常执行

**注意事项**：
- 如果按钮长时间灰化，检查是否有错误
- 查看日志区域了解加载进度
- 不要强制关闭程序

### 列表编号清理

**自动处理**：
- 无需手动配置
- 自动检测带编号的列表
- 自动清理并添加符号

**支持的格式**：
- `1、`、`2、`、`3、`
- `1)`、`2)`、`3)`
- `(1)`、`(2)`、`(3)`
- 以及其他常见格式
