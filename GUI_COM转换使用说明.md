# GUI 界面使用 Word COM 转换 - 完整保留 Visio 图

## 🎉 新功能

现在 GUI 界面支持 **Word COM 转换模式**，可以完整保留所有 Visio 图和 OLE 对象！

## 🚀 使用方法

### 步骤 1：启动 GUI

```bash
python doc_converter_gui.py
```

或者双击 `启动转换工具.bat`

### 步骤 2：选择文件

1. 点击"浏览"选择源文件（如 `8js3.docx`）
2. 点击"浏览"选择模板文件（如 `mb.docx`）

### 步骤 3：**勾选"使用 Word COM 转换"** ⭐

在"选项设置"区域，您会看到两个复选框：

- ☐ 进行祈使语气转换
- ☑ **使用 Word COM 转换（保留 Visio 图）** ← **勾选这个！**

### 步骤 4：开始转换

点击"开始转换"按钮。

## 📊 两种模式对比

| 特性 | 普通模式（未勾选） | Word COM 模式（勾选） |
|------|-------------------|---------------------|
| Visio 图复制 | ❌ 插入占位文本 | ✅ **完整保留** |
| OLE 对象 | ❌ 插入占位文本 | ✅ **完整保留** |
| 样式应用 | ✅ 精细控制 | ⚠️ 依赖 Word |
| 祈使语气转换 | ✅ 支持 | ❌ 不支持* |
| 应答句插入 | ✅ 支持 | ❌ 不支持* |
| 速度 | 快 | 较慢（约 1-2 分钟） |
| 需要 Word | ❌ 不需要 | ✅ 需要安装 |

\* 如果需要这些功能，可以先用 COM 转换，然后再用普通模式处理。

## 💡 推荐工作流程

### 场景 1：只需要保留 Visio 图

1. 勾选"使用 Word COM 转换"
2. 点击"开始转换"
3. 完成！所有 Visio 图都保留了

### 场景 2：需要保留 Visio 图 + 应用样式 + 其他处理

**两步走策略**：

#### 第一步：COM 转换（保留 Visio 图）
1. 勾选"使用 Word COM 转换"
2. 取消勾选"进行祈使语气转换"
3. 点击"开始转换"
4. 生成 `8js3_converted.docx`

#### 第二步：普通转换（应用样式和其他处理）
1. **取消勾选**"使用 Word COM 转换"
2. 选择第一步生成的文件作为源文件
3. 勾选需要的功能（祈使语气、应答句等）
4. 点击"开始转换"
5. 生成最终文件

## ❓ 常见问题

### Q1: 为什么我勾选了还是看到占位文本？

**A**: 请确认：
1. 确实勾选了"使用 Word COM 转换（保留 Visio 图）"
2. 查看日志中是否有 "⚠ 使用 Word COM 转换模式" 的提示
3. 如果没有，可能是旧版本的 GUI，请重新启动

### Q2: 转换后 Word 进程还在运行？

**A**: 正常情况下程序会自动关闭 Word。如果异常退出：
- 打开任务管理器
- 找到 `WINWORD.EXE`
- 结束进程

### Q3: 可以同时转换多个文件吗？

**A**: 可以！COM 模式支持批量转换。
- 添加多个源文件
- 勾选"使用 Word COM 转换"
- 点击"开始转换"
- 程序会依次处理每个文件

### Q4: COM 模式能应用模板样式吗？

**A**: COM 模式的"另存为"方式会保留原始样式。
如果需要应用新模板的样式：
1. 先用 COM 转换保留 Visio 图
2. 再用普通模式应用样式（Visio 图会被替换为占位文本）
3. 手动从第一步的文件中复制 Visio 图到最终文件

或者等待未来版本实现完整的 COM + 模板样式应用。

### Q5: 速度慢怎么办？

**A**: COM 模式确实比普通模式慢，因为要启动 Word。
- 单个文件：约 1-2 分钟
- 批量处理：每个文件约 1-2 分钟
- 建议：耐心等待，这是为了保证 Visio 图完整性
