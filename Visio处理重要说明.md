# Visio 图处理 - 重要说明

## ⚠️ 当前状态

经过深入分析和测试，我们发现 **OLE 对象（包括 Visio 图）的完整复制是一个非常复杂的技术问题**。

### 问题原因

OLE 对象在 Word 文档中的存储涉及多个层面：
1. XML 引用（document.xml）
2. 关系 ID 映射（document.xml.rels）
3. 嵌入的二进制文件（word/embeddings/）
4. 可能的缩略图和预览

简单地复制 XML 会导致：
- ❌ 关系 ID 冲突
- ❌ 嵌入文件丢失
- ❌ 文档损坏
- ❌ Word 无法打开文件

## ✅ 当前实现

为了避免文档损坏，我们采用了**安全的降级策略**：

### 处理方式

当检测到包含 Visio 图或 OLE 对象的段落时：
1. **跳过复制**该对象
2. **插入占位文本**：`[此处有 Visio 图或 OLE 对象，请手动复制]`
3. **记录警告信息**，告知用户哪些段落包含特殊对象
4. **继续处理**其他内容，确保文档不损坏

### 优点

- ✅ **保证文档完整性** - 生成的文件可以正常打开
- ✅ **清晰的提示** - 用户知道哪里需要手动处理
- ✅ **稳定的性能** - 不会因为大量 OLE 对象导致程序卡死

### 缺点

- ❌ 需要手动复制 Visio 图
- ❌ 失去自动化优势

## 📋 使用建议

### 方案 1：手动复制（推荐用于少量 Visio 图）

1. 运行转换程序
2. 查看警告信息，了解哪些段落包含 Visio 图
3. 打开源文件和转换后的文件
4. 手动复制粘贴 Visio 图到对应位置

### 方案 2：使用 Word COM 自动化（Windows，高级用户）

如果您熟悉 Python 和 Windows COM，可以使用以下方法：

```python
import win32com.client

# 启动 Word
word = win32com.client.Dispatch("Word.Application")
word.Visible = False

# 打开文档
source_doc = word.Documents.Open("source.docx")
target_doc = word.Documents.Open("template.docx")

# 使用 Word API 复制内容
# ... (需要自行实现)

# 保存并关闭
target_doc.SaveAs("output.docx")
source_doc.Close()
target_doc.Close()
word.Quit()
```

## 💡 最佳实践

### 对于包含少量 Visio 图的文档
- 使用当前版本转换
- 手动复制 5-10 个 Visio 图
- 总耗时约 5-10 分钟

### 对于包含大量 Visio 图的文档（如 8js3.docx）
- 考虑是否真的需要保留所有 Visio 图的可编辑性
- 如果不需要，可以先将 Visio 图导出为图片
- 或者分批处理，每次处理一部分
