# 快速开始指南

## 立即使用

### 方法1：双击批处理文件（最简单）
直接双击 `启动转换工具.bat` 即可运行程序。

### 方法2：命令行启动
```bash
python doc_converter_gui.py
```

## 核心功能说明

### 需求1：文件选择
- **单文件**：点击"浏览..."选择一个源文档
- **多文件**：点击"多选..."选择多个源文档

### 需求2：样式识别与映射
程序会自动：
1. 识别源文档中使用的所有样式（显示在左侧列表）
2. 识别模板文档中的所有可用样式（显示在右侧列表）
3. **单文件模式**：可勾选"使用自定义样式映射"，手动指定每个源样式对应的模板样式
4. **多文件模式**：使用系统默认转换规则

### 需求3&4：样式转换
- 将源文档样式转换为模板文档样式
- 新文档只保留模板文档的格式样式
- 不会将源文档的样式引入新文档

### 需求5：祈使语气转换
自动将祈使语气转换为投标人语气：
- 必须 → 将
- 应/须 → 将  
- 不得/不应/不可 → 不会
- 不能 → 无法
- 投标人 → 本投标人

智能跳过例外词（如"响应"、"应用"、"不可抗力"等）

### 需求6：插入应答句
- 选择你需要插入应答句的位置
- 默认内容："应答：本投标人理解并满足要求。"
- 可在界面中自定义应答文本和样式

### 需求7：基于现有脚本
已整合您提供的三个脚本：
- `3.convert_styles_fh_v4_fixtable.py` - 样式转换
- `5.deal_with_mood.py` - 语气转换
- `6.insert_response_paragraphs.py` - 插入应答句

## 使用流程示例

### 场景：转换投标文件

1. **准备文件**
   - 源文档：投标技术方案.docx
   - 模板文档：标准格式模板.docx

2. **操作步骤**
   ```
   1. 点击"浏览..."选择"投标技术方案.docx"
   2. 点击"浏览..."选择"标准格式模板.docx"
   3. 确认"进行祈使语气转换"已勾选
   4. （可选）如需自定义样式映射，勾选"使用自定义样式映射"
   5. （可选）修改应答句文本
   6. 点击"开始转换"
   7. 等待转换完成
   ```

3. **查看结果**
   - 输出文件：投标技术方案_converted.docx
   - 日志区域显示详细转换过程
   - 如有错误会生成 xxx_err.log 文件

## 批量转换

同时转换多个投标文件：
1. 点击"多选..."选择所有源文档
2. 选择模板文档
3. 点击"开始转换"
4. 每个文件会生成独立的输出文件

## 常见问题

**Q: 如何知道哪些样式被转换了？**  
A: 查看界面中的"样式信息"区域，左侧显示源文档样式，右侧显示模板文档样式。

**Q: 转换后格式不对怎么办？**  
A: 使用"自定义样式映射"功能，手动指定每个样式的对应关系。

**Q: 可以取消某些转换步骤吗？**  
A: 可以取消勾选"进行祈使语气转换"来跳过语气转换步骤。

**Q: 输出文件在哪里？**  
A: 默认在源文件同目录下，文件名为"原文件名_converted.docx"。也可通过"浏览..."指定输出路径。

## 技术支持

如遇问题，请检查：
1. Python版本是否为3.7+
2. 依赖包是否正确安装
3. 源文件和模板文件是否存在且可访问
4. 查看日志区域的错误信息
5. 查看生成的 xxx_err.log 文件
