# 📱 配置个人微信收款码指南

## ✅ 功能说明

现在系统支持使用您的**个人微信收款码**进行充值！

### 工作流程

```
1. 用户选择充值档位
         ↓
2. 系统显示您的个人收款码
         ↓
3. 用户微信扫码支付
         ↓
4. 您收到款项（微信通知）
         ↓
5. 用户点击"✅ 我已支付"按钮
         ↓
6. 系统自动充值到用户账户
         ↓
7. 用户余额实时更新
```

---

## 🔧 配置步骤

### 步骤1：获取您的微信收款码

1. 打开微信
2. 点击右上角"+" → "收付款"
3. 选择"二维码收款"
4. 点击"保存收款码"
5. 将图片保存到电脑

### 步骤2：重命名并放置文件

1. 将收款码图片重命名为：`personal_qr_code.png`
2. 放到项目根目录：`E:\LingMa\WordStyle\personal_qr_code.png`

**或者修改代码中的路径**：

编辑 `app.py` 第507行：
```python
qr_code_path = "personal_qr_code.png"  # 修改为您的图片路径
```

例如：
```python
qr_code_path = "D:/MyDocuments/wechat_qr.png"
```

### 步骤3：重启应用

```bash
# 重启Streamlit应用
cd e:\LingMa\WordStyle
.venv\Scripts\python.exe -m streamlit run app.py --server.port=8501
```

## 💡 重要提示

### ⚠️ 安全性说明

**当前实现的特点**：
- ✅ 简单直接，无需商户号
- ✅ 用户扫码后手动确认
- ❌ **无法自动验证是否真的支付**
- ❌ 依赖用户诚实点击"我已支付"

**适用场景**：
- ✅ 小规模、信任度高的用户群体
- ✅ 朋友、同事之间使用
- ✅ 测试和演示环境

**不适用场景**：
- ❌ 大规模公开服务
- ❌ 需要严格防作弊的场景
- ❌ 高金额交易

---

### 🔒 如何防止作弊？

#### 方案1：人工审核（推荐）

1. 用户点击"我已支付"
2. 系统记录待审核订单
3. 您查看微信收款通知
4. 手动在后台确认充值

**优点**：完全可靠  
**缺点**：需要人工操作

---

#### 方案2：支付凭证上传

1. 用户支付后截图
2. 上传支付成功截图
3. 系统OCR识别金额
4. 自动或人工审核

**优点**：半自动化  
**缺点**：实现复杂

---

#### 方案3：固定金额快速通道

为每个充值档位生成固定的收款码：
- ¥10 收款码
- ¥50 收款码
- ¥100 收款码

用户扫描对应金额的收款码，系统更容易核对。

## 📝 常见问题

### Q1: 用户没支付就点击"我已支付"怎么办？

**A**: 当前系统无法防止这种情况。建议：
- 在小范围信任群体中使用
- 定期检查充值记录和实际收款
- 发现异常时手动调整余额

---

### Q2: 可以支持多个收款码吗？

**A**: 可以！修改代码：

```python
# 根据金额显示不同的收款码
if amount == 10:
    qr_code_path = "qr_10.png"
elif amount == 50:
    qr_code_path = "qr_50.png"
elif amount == 100:
    qr_code_path = "qr_100.png"
```

---

### Q3: 如何查看充值记录？

**A**: 查看 `user_data.json` 文件：

```json
{
  "abc123def456": {
    "balance": 160.0,
    "paragraphs_remaining": 160000,
    "recharge_history": [
      {
        "time": "2026-05-01 12:00:00",
        "amount": 10.0,
        "paragraphs": 10000,
        "order_id": "PAY_20260501120000",
        "method": "wechat_scan"
      }
    ]
  }
}
```

---

### Q4: 可以撤销充值吗？

**A**: 当前不支持。如需撤销，手动编辑 `user_data.json` 文件。
