# 列表符号配置说明

## 📝 概述

文档转换工具支持**自定义列表段落符号**，用户可以根据需要在 GUI 界面中手动配置。

## 🎯 默认值更新（v2.4）

- **改进前**：`■ `（黑色方块）
- **改进后**：`● `（黑色圆点）
- **原因**：圆点更美观，符合大多数文档规范

## ⚙️ 如何配置列表符号

### 在 GUI 中配置

1. **启动 GUI**
   ```bash
   python doc_converter_gui.py
   ```

2. **找到配置区域**
   - 滚动到"样式信息"区域
   - 看到"列表段落符号:"输入框

3. **输入自定义符号**
   - 点击输入框
   - 输入您喜欢的符号
   - 建议符号后加一个空格

4. **开始转换**
   - 配置完成后点击"开始转换"
   - 转换后的文档将使用您配置的符号

### 配置示例

| 符号 | 效果 | 适用场景 |
|------|------|----------|
| `● ` | ● 项目一 | 通用文档（默认） |
| `- ` | - 项目一 | Markdown 风格 |
| `* ` | * 项目一 | 简洁风格 |
| `→ ` | → 项目一 | 流程说明 |
| `■ ` | ■ 项目一 | 技术文档 |
| `◆ ` | ◆ 项目一 | 强调重点 |
| `>> ` | >> 项目一 | 引用内容 |
| `-> ` | -> 项目一 | 代码注释风格 |

## 💡 使用建议

### 推荐配置

**一般文档**：
- 使用默认的 `● `
- 美观、专业、易读

**技术文档**：
- 使用 `- ` 或 `* `
- 简洁、清晰

**演示文稿**：
- 使用 `→ ` 或 `◆ `
- 醒目、有层次感

**代码相关**：
- 使用 `>> ` 或 `-> `
- 符合编程习惯

### 注意事项

1. **符号后加空格**
   - ✅ 推荐：`● `（有空格）
   - ❌ 不推荐：`●`（无空格）
   - 原因：有空格排版更美观

2. **可以使用多个字符**
   - ✅ 支持：`>> `、`-> `、`==> `
   - 适合特殊场景

3. **仅对当前转换生效**
   - 每次转换可以配置不同的符号
   - 不会保存为全局设置
   - 下次启动 GUI 会恢复默认值 `● `

4. **特殊字符支持**
   - 支持 Unicode 字符
   - 如：`★ `、`♥ `、`✓ ` 等
   - 确保字体支持该字符

### 应用位置

列表符号会在以下场景自动添加：

1. **带编号的段落**
   - 检测到 Word 自动编号
   - 移除自动编号
   - 添加配置的符号

2. **表格中的列表**
   - 表格单元格内的编号段落
   - 同样应用配置的符号

## ❓ 常见问题

### Q1: 为什么默认值改为 ● ？

**A**: 
- ● 更符合现代文档规范
- 视觉效果更柔和
- 在打印和屏幕显示上都更清晰

### Q2: 可以保存自定义设置为永久默认值吗？

**A**: 
- 当前版本不支持
- 每次启动 GUI 都会恢复为 `● `
- 如需永久修改，可以编辑 `doc_converter_gui.py` 第 196 行

### Q3: 如果输入空字符串会怎样？

**A**: 
- 列表项前面不会有符号
- 但仍然会清理编号
- 不推荐这样做，会影响可读性

### Q4: 符号会影响样式映射吗？

**A**: 
- 不会
- 符号只是添加到文本前面
- 不影响段落的样式属性

### Q5: 可以在命令行中指定列表符号吗？

**A**: 
- 当前版本不支持命令行参数
- 只能在 GUI 中配置
- 如需命令行支持，可以提交功能请求
