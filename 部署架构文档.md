## 🏗️ 当前部署架构

### 架构概览

WordStyle 项目采用**前后端分离架构**，通过 **API 模式**实现数据访问。

```
┌─────────────────────────┐         HTTP API         ┌─────────────────────────
│   前端 (Streamlit Cloud) │ ──────────────────────> │   后端 (Render)          │
│                          │                         │                          │
│  • 用户界面              │ <────────────────────── │  • FastAPI 服务          │
│  • 文档上传              │     JSON 响应           │  • 数据库操作            │
│  • 转换配置              │                         │  • 业务逻辑              │
│  • 结果下载              │                         │                          │
└─────────────────────────┘                         └──────────┬──────────────┘
                                                               │
                                                               │ PostgreSQL 连接
                                                               ▼
                                                    ┌─────────────────────────┐
                                                    │   数据库 (Supabase)      │
                                                    │                          │
                                                    │  • users 表             │
                                                    │  • orders 表            │
                                                    │  • conversion_tasks 表  │
                                                    │  • system_config 表     │
                                                    ─────────────────────────┘
```

### 部署平台

| 组件 | 平台 | URL | 说明 |
|------|------|-----|------|
| **前端** | Streamlit Cloud | https://wordstyle.streamlit.app | 用户界面，文档转换功能 |
| **管理后台** | Streamlit Cloud | https://wordstyle-manage.streamlit.app | 管理员界面，用户和任务管理 |
| **后端 API** | Render | https://wordstyle-backend.onrender.com | FastAPI 服务，数据库操作 |
| **数据库** | Supabase | PostgreSQL | 用户数据、订单、任务存储 |

## 🔌 数据源模式

### 三种数据源模式

系统支持三种数据源模式，通过配置文件自动检测：

#### 1. **API 模式**（当前生产环境）

**触发条件**：`BACKEND_URL` 存在 + `USE_SUPABASE=true`

**使用场景**：Streamlit Cloud 等受限环境（不允许出站数据库连接）

**数据流**：
```
前端 → HTTP API 调用 → 后端 FastAPI → 数据库操作 → 返回结果
```

**配置**：
```toml
# .streamlit/secrets.toml (Streamlit Cloud)
BACKEND_URL = "https://wordstyle-backend.onrender.com"
USE_SUPABASE = true
```

#### 2. **Supabase 模式**（Render 部署）

**触发条件**：只有 `DATABASE_URL` + `USE_SUPABASE=true`

**使用场景**：Render 等允许出站数据库连接的环境

**数据流**：
```
前端 → 直接连接 Supabase (SQLAlchemy ORM) → 数据库操作
```

**配置**：
```env
# Render 环境变量
DATABASE_URL = "postgresql://..."
USE_SUPABASE = true
```

#### 3. **Local 模式**（本地开发）

**触发条件**：未配置 `USE_SUPABASE` 或 `BACKEND_URL`

**使用场景**：本地开发测试

**数据流**：
```
前端 → SQLite (conversion_tasks.db) + JSON (user_data.json)
```

## 🔄 前后端分离架构

### 前端 (Streamlit Cloud)

**职责**：
- 用户界面渲染
- 文件上传与下载
- 转换配置管理
- 通过 HTTP API 调用后端服务

**关键文件**：
```
e:\LingMa\WordStyle/
├── app.py                      # 主应用（用户端）
├── admin_web.py                # 管理后台
├── data_manager.py             # 统一数据访问层
├── config.py                   # 配置管理
├── user_manager.py             # 用户管理（本地模式）
├── task_manager.py             # 任务管理（本地模式）
└── requirements.txt            # 依赖（包含后端依赖）
```

**数据访问层**：`data_manager.py` 根据 `DATA_SOURCE` 环境变量自动选择数据源：
- `api` 模式 → 调用后端 API
- `supabase` 模式 → 直接连接数据库
- `local` 模式 → SQLite + JSON

### 后端 (Render)

**职责**：
- 提供 RESTful API 接口
- 数据库 CRUD 操作
- 业务逻辑处理
- 用户认证与授权

**关键文件**：
```
e:\LingMa\WordStyle\backend/
── main.py                     # FastAPI 应用入口
├── app/
│   ├── api/
│   │   ├── auth.py            # 认证接口
│   │   ├── users.py           # 用户接口
│   │   ├── orders.py          # 订单接口
│   │   ├── tasks.py           # 任务接口
│   │   └── admin.py           # 管理员接口
│   ├── core/
│   │   ├── config.py          # 后端配置
│   │   └── database.py        # 数据库连接
│   ├── models.py              # SQLAlchemy 模型
│   └── schemas.py             # Pydantic 数据模型
└── requirements.txt           # 后端依赖
```

**API 端点**：

| 端点 | 方法 | 功能 | 说明 |
|------|------|------|------|
| `/api/health` | GET | 健康检查 | 返回服务状态 |
| `/api/users/{user_id}` | POST | 创建/更新用户 | 前端写入用户数据 |
| `/api/users` | GET | 获取所有用户 | 管理后台使用 |
| `/api/orders` | POST | 创建订单 | 充值功能 |
| `/api/tasks` | GET | 获取任务列表 | 管理后台使用 |
| `/api/tasks/{task_id}` | GET | 获取任务状态 | 前端查询转换进度 |

## ⚙️ 部署平台配置

### Streamlit Cloud 配置

**Secrets 配置**（`.streamlit/secrets.toml`）：

```toml
# 后端 API 地址
BACKEND_URL = "https://wordstyle-backend.onrender.com"

# 启用 Supabase 模式
USE_SUPABASE = true

# 数据库连接（API 模式下不需要，但保留用于切换模式）
DATABASE_URL = "postgresql://..."
```

**依赖配置**（根目录 `requirements.txt`）：

⚠️ **重要**：必须包含后端依赖，因为管理后台会导入后端模块。

```txt
streamlit>=1.28.0
python-docx>=1.0.0
lxml>=4.9.0
requests>=2.31.0
sqlalchemy>=2.0.0
psycopg2-binary>=2.9.0
pydantic>=2.0.0
python-dotenv>=1.0.0
```

### Render 配置

**render.yaml**（自动部署配置）：

```yaml
services:
  - type: web
    name: wordstyle-backend
    env: python
    plan: free
    buildCommand: pip install -r backend/requirements.txt
    startCommand: cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: DATABASE_URL
        fromDatabase:
          name: wordstyle-db
          property: connectionString
      - key: PYTHON_VERSION
        value: "3.11.0"
```

**数据库配置**：

```yaml
databases:
  - name: wordstyle-db
    plan: free
```

## 🗄️ 数据库表结构

### users 表（用户信息）

```sql
CREATE TABLE users (
    id VARCHAR(12) PRIMARY KEY,          -- 12位字符串ID（应用层生成）
    balance FLOAT DEFAULT 0.0,           -- 账户余额（元）
    paragraphs_remaining INTEGER DEFAULT 0,  -- 剩余段落数
    total_paragraphs_used INTEGER DEFAULT 0, -- 累计使用段落数
    total_converted INTEGER DEFAULT 0,   -- 累计转换文件数
    is_active BOOLEAN DEFAULT TRUE,      -- 是否激活
    created_at TIMESTAMPTZ DEFAULT NOW(), -- 创建时间
    last_login TIMESTAMPTZ,              -- 最后登录时间
    updated_at TIMESTAMPTZ               -- 更新时间
);
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VARCHAR(12) | 用户唯一标识，12位 MD5 哈希字符串 |
| `balance` | FLOAT | 账户余额（元），默认 0.0 |
| `paragraphs_remaining` | INTEGER | 剩余可用段落数 |
| `total_paragraphs_used` | INTEGER | 累计已用段落数 |
| `total_converted` | INTEGER | 累计转换文件数 |
| `is_active` | BOOLEAN | 用户是否激活 |
| `created_at` | TIMESTAMPTZ | 用户创建时间 |
| `last_login` | TIMESTAMPTZ | 最后登录时间 |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 |

**索引**：
- 主键索引：`users_pkey` on `id`

### orders 表（订单信息）

```sql
CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_no VARCHAR(64) UNIQUE NOT NULL,  -- 订单号
    user_id VARCHAR(12) NOT NULL,          -- 用户ID（外键）
    amount FLOAT NOT NULL,                 -- 订单金额
    paragraphs INTEGER NOT NULL,           -- 购买段落数
    status VARCHAR(20) DEFAULT 'PENDING',  -- 订单状态
    created_at TIMESTAMPTZ DEFAULT NOW(),
    paid_at TIMESTAMPTZ,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### conversion_tasks 表（转换任务）

```sql
CREATE TABLE conversion_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id VARCHAR(64) UNIQUE NOT NULL,   -- 任务ID
    user_id VARCHAR(12) NOT NULL,          -- 用户ID（外键）
    filename VARCHAR(255),                 -- 文件名
    paragraphs INTEGER,                    -- 段落数
    cost FLOAT,                            -- 费用
    status VARCHAR(20) DEFAULT 'PENDING',  -- 任务状态
    progress INTEGER DEFAULT 0,            -- 进度（0-100）
    output_files TEXT,                     -- 输出文件列表（JSON）
    error_message TEXT,                    -- 错误信息
    created_at TIMESTAMPTZ DEFAULT NOW(),
    completed_at TIMESTAMPTZ,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### system_config 表（系统配置）

```sql
CREATE TABLE system_config (
    id SERIAL PRIMARY KEY,
    config_key VARCHAR(100) UNIQUE NOT NULL,
    config_value TEXT NOT NULL,
    description VARCHAR(500),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 🔄 数据流程

### 用户首次访问流程

```
1. 用户打开 Streamlit Cloud 应用
   ↓
2. app.py 初始化
   - 检查 session_state 中是否有 user_id
   - 如无，生成新的 12 位 user_id（基于 hostname 哈希）
   ↓
3. 调用 data_manager.load_user_data(user_id)
   - 在 API 模式下，调用后端 API: GET /api/users/{user_id}
   ↓
4. 调用 claim_free_paragraphs(user_id)
   - 检查是否是今日首次访问
   - 如是，赠送 10000 免费段落
   ↓
5. 调用 register_or_login_user(user_id, user_data)
   - 在 API 模式下，调用后端 API: POST /api/users/{user_id}
   - 后端执行 SQL: INSERT INTO users (...)
   ↓
6. 用户数据写入 Supabase 数据库
   - id: 12位字符串
   - balance: 0.0
   - paragraphs_remaining: 10000
   - total_converted: 0
   - is_active: True
   - created_at: NOW()
   - last_login: NOW()
```

### 文档转换流程

```
1. 用户上传文档并点击"开始转换"
   ↓
2. 前端调用后端 API 创建任务
   - POST /api/tasks
   - 请求体: {user_id, filename, paragraphs, cost}
   ↓
3. 后端创建任务记录
   - INSERT INTO conversion_tasks (...)
   - 返回 task_id
   ↓
4. 前端轮询任务状态
   - GET /api/tasks/{task_id}
   - 每 2 秒查询一次
   ↓
5. 后端执行转换（异步）
   - 更新任务状态: PENDING → PROCESSING → COMPLETED
   - 更新进度: 0 → 100
   ↓
6. 转换完成
   - 更新用户数据: paragraphs_remaining -= paragraphs
   - 更新用户统计: total_converted += 1
   ↓
7. 前端显示下载链接
```

### 用户数据更新流程

```
1. 用户完成文档转换
   ↓
2. 前端调用后端 API 更新用户数据
   - POST /api/users/{user_id}
   - 请求体: {
       balance: 0.0,
       paragraphs_remaining: 9900,
       total_paragraphs_used: 100,
       total_converted: 1
     }
   ↓
3. 后端执行 SQL 更新
   - UPDATE users SET 
       paragraphs_remaining = 9900,
       total_paragraphs_used = 100,
       total_converted = 1,
       last_login = NOW()
     WHERE id = '{user_id}'
   ↓
4. 返回更新结果
```

## 🔍 故障排查

### 前端无法连接后端

**症状**：管理后台显示"连接失败"或"无法加载数据"

**排查步骤**：
1. 检查 Streamlit Cloud Secrets 中 `BACKEND_URL` 是否正确
2. 访问 `https://wordstyle-backend.onrender.com/api/health` 检查后端是否运行
3. 查看 Render 控制台日志，确认服务状态
4. 检查 Render 服务是否因免费套餐限制而休眠

### 用户数据无法写入数据库

**症状**：管理后台用户数为 0，但前端有用户访问

**排查步骤**：
1. 检查 `users` 表 `id` 字段类型是否为 `VARCHAR(12)`
2. 检查后端日志是否有 SQL 错误
3. 运行测试脚本：`cd backend && python test_new_user_flow.py`
4. 检查 Supabase 数据库连接是否正常

### 前后端版本不一致

**症状**：API 调用失败，返回 422 或 500 错误

**排查步骤**：
1. 检查前端和后端代码是否都已更新到最新版本
2. 确认 `data_manager.py` 中的 API 调用路径与后端路由一致
3. 检查 `schemas.py` 中的数据模型是否匹配
4. 查看后端日志中的具体错误信息

##  维护指南

### 更新前端

1. 修改代码后提交到 Git
2. Streamlit Cloud 自动检测并重新部署（或手动触发）
3. 验证功能正常

### 更新后端

1. 修改 `backend/` 目录下的代码
2. 提交到 Git（确保包含 `render.yaml`）
3. Render 自动检测并重新部署
4. 查看 Render 控制台日志确认部署成功
5. 访问 `/api/health` 验证服务运行

### 数据库迁移

使用 Alembic 进行数据库迁移：

```bash
cd backend

# 创建新迁移
alembic revision -m "description"

# 执行迁移
alembic upgrade head

# 回滚迁移
alembic downgrade -1
```

迁移文件位置：`backend/alembic/versions/`
