# 文件保存逻辑修复说明 - v2.6.1

## 🔧 本次修复内容

### 问题描述

在 v2.6 版本中，虽然改进了文件被占用时的保存逻辑（使用时间戳后缀），但存在两个问题：

1. **输出文件列表显示错误**：列表中显示的是原始文件名，而不是实际保存的文件名（带时间戳）
2. **时间戳格式过长**：`_YYYYMMDD_HHMMSS` 格式太长，用户希望简化为 `_HHMM`

### 修复方案

#### 1. ✅ 修复输出文件列表显示

**问题根源**：
- `full_convert()` 返回的是固定消息，没有返回实际文件名
- GUI 在调用前就添加了原始文件名到列表
- 当文件被占用时，实际保存的文件名不同，但列表未更新

**解决方案**：
- 修改 `insert_response_after_headings()` 返回实际文件名
- 修改 `full_convert()` 捕获并返回实际文件名
- 修改 GUI 调用，使用返回的实际文件名更新列表

**代码流程**：
```
save_with_retry() 
  → 返回 (success, actual_file, msg)
  
insert_response_after_headings()
  → 返回 (success, actual_file, msg)
  
full_convert()
  → 返回 (success, actual_file, msg)
  
GUI do_conversion()
  → 捕获 actual_file
  → 添加到 output_files 列表
  → 显示 basename(actual_file)
```

#### 2. ✅ 简化时间戳格式

**改进前**：
```
document_converted_20260425_143025.docx
```

**改进后**：
```
document_converted_1430.docx
```

**优势**：
- 更简洁
- 更易读
- 符合用户习惯

## 🔧 技术实现

### 修改的文件

1. **doc_converter.py**
   - 修改 `insert_response_after_headings()` 返回值
   - 修改 `full_convert()` 捕获和返回实际文件名
   - 简化时间戳格式为 `_HHMM`

2. **doc_converter_gui.py**
   - 修改单文件模式的调用，捕获实际文件名
   - 修改多文件模式的调用，捕获实际文件名
   - 使用 `os.path.basename()` 显示文件名

### 关键代码变更

#### 1. insert_response_after_headings 返回值

```python
# 修复前
return True, f"已插入 {insert_count} 个应答句..."

# 修复后
return True, actual_file, f"已插入 {insert_count} 个应答句..."
```

#### 2. full_convert 捕获实际文件名

```python
# 修复前
success, msg = self.insert_response_after_headings(...)
return True, "转换成功完成！"

# 修复后
actual_output_file = output_file
success, actual_file, msg = self.insert_response_after_headings(...)
actual_output_file = actual_file
return True, actual_output_file, "转换成功完成！"
```

#### 3. GUI 使用实际文件名

```python
# 修复前
success, msg = self.converter.full_convert(...)
if success:
    self.output_files.append(output)
    self.output_listbox.insert(END, output)

# 修复后
success, actual_file, msg = self.converter.full_convert(...)
if success:
    self.output_files.append(actual_file)
    self.output_listbox.insert(END, os.path.basename(actual_file))
```

#### 4. 时间戳格式简化

```python

# 修复后

```

## ⚠️ 注意事项

### 时间戳精度

**当前格式**：`_HHMM`（时分）

**优点**：
- 简洁易读
- 适合大多数场景

**潜在问题**：
- 同一分钟内多次保存可能冲突
- 如果频繁转换，可能需要手动清理旧文件

**建议**：
- 如果需要更高精度，可以改回 `_HHMMSS` 或 `_YYYYMMDD_HHMMSS`
- 定期清理带时间戳的备份文件

### 文件管理

**命名规律**：
```
document_converted.docx          ← 原始预期文件名
document_converted_1430.docx     ← 14:30 保存的
document_converted_1431.docx     ← 14:31 保存的
document_converted_1432.docx     ← 14:32 保存的
```

**管理建议**：
1. 保留最新的文件
2. 删除旧的时间戳文件
3. 可以使用搜索功能：`*_????_.docx`
